# example

## Git Contribution submission specification

- `build` 빌드
- `chore` 업데이트/스캐폴드 구성 수정 등에 의존
- `ci` 지속적인 통합
- `docs` 문서/주석
- `feat` 새로운 기능을 추가
- `fix` 문제/BUG를 수정
- `perf` 최적화/성능 향상
- `refactor` 리팩터링
- `revert` 변경 내용을 실행 취소
- `style` 코드 스타일 관련은 실행 결과에 영향을 주지 않습니다
- `test` 테스트 관련
- `wip` 개발 중
